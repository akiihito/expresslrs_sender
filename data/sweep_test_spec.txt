sweep_test.csv 仕様
====================

概要: CH5 を ARM に切り替えた後、CH1-CH4 を正弦波で一巡させるテストデータ

フレーム数: 601
総時間:     12秒
間隔:       20ms (50Hz ソースレート、500Hz で再生)
チャンネル: 16ch (CH9-CH16 は固定値)

フェーズ構成
------------

Phase 1: 待機 (0-1000ms, 1秒)
  CH1-CH4: 中央値 (CH3=172)
  CH5: 172 (DISARM)
  目的: システム安定化

Phase 2: Arm 遅延 (1000-4000ms, 3秒)
  CH1-CH4: 中央値 (CH3=172)
  CH5: 1811 (ARM)
  目的: safety_monitor の arm_delay_ms (3000ms) を待つ
        この間 CH3 は safety_monitor により 172 に強制される

Phase 3: スイープ (4000-10000ms, 6秒, 1サイクル)
  CH1 (Roll):     992 + 819.5 * sin(2πt/6s)       全範囲 172-1811
  CH2 (Pitch):    992 + 819.5 * cos(2πt/6s)       全範囲 172-1811 (CH1より90°先行)
  CH3 (Throttle): 172 + 400 * sin(πt/6s)          控えめ 172-572 (範囲の約25%)
  CH4 (Yaw):      992 - 819.5 * sin(2πt/6s)       全範囲 172-1811 (CH1と逆位相)
  CH5: 1811 (ARM)

Phase 4: 復帰 (10000-11000ms, 1秒)
  CH1-CH4: 中央値 (CH3=172)
  CH5: 1811 (ARM)
  目的: スティックを中央に戻す

Phase 5: Disarm (11000-12000ms, 1秒)
  CH1-CH4: 中央値 (CH3=172)
  CH5: 172 (DISARM)
  目的: 安全に停止

チャンネル別の動き
------------------

CH1 (Roll):     ─────┐  sin波で右→中央→左→中央 (1サイクル)
CH2 (Pitch):    ─────┐  cos波で前→中央→後→中央 (CH1より90°先行)
CH3 (Throttle): ─────┐  半正弦で 172→572→172 (安全な低スロットル)
CH4 (Yaw):      ─────┐  CH1の逆位相で左→中央→右→中央
CH5 (Arm):      LOW──┐HIGH──────────────────────┐LOW──

CRSF 値の範囲
--------------

最小値: 172  (CRSF_CHANNEL_MIN)
中央値: 992  (CRSF_CHANNEL_MID)
最大値: 1811 (CRSF_CHANNEL_MAX)

実行方法
--------

sudo ./expresslrs_sender play -H data/sweep_test.csv
